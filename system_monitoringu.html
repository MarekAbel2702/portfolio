System Monitoringu Logów

System analizy i monitoringu logów aplikacji

Opis Projektu

Projekt przedstawia kompletny system analizy danych z plików logów. Rozwiązanie umożliwia automatyczne przetwarzanie logów aplikacyjnych zapisanych w pliku tekstowym, ich strukturalne zapisanie w relacyjnej bazie danych oraz eksport i prezentację w różnych formatach i formach graficznych. System składa się z kilku niezależnych modułów, które odpowiadają za poszczególne etapy: parsowanie danych, komunikację z bazą, eksport oraz analizę i wizualizację. Projekt został zaprojektowany z naciskiem na czytelność, rozszerzalność i praktyczne zastosowanie.

Użyte Technologie

  • Python
  • SQL
  • Matplotlib
  • Pandas
  • Seaborn
  • Git

📝 Zakres prac:

  • ✅ Projektowanie bazy danych: zaprojektowałem relacyjną strukturę (tabela Logs), umożliwiającą przechowywanie danych z logów w MS SQL Server.
  • ✅ Parsowanie logów: stworzyłem parser w Pythonie, który rozpoznaje strukturę wpisów logów (data, poziom, wiadomość) z plików tekstowych.
  • ✅ Integracja z bazą danych: połączenie Pythona z MS SQL Server przy użyciu pyodbc – dane z logów zapisywane są automatycznie do bazy.
  • ✅ Eksport danych: zaimplementowałem moduł eksportujący dane z bazy do plików w formatach (CSV, JSON, XML, HTML)
  • ✅ Wizualizacja danych: przygotowałem graficzną analizę danych przy użyciu matplotlib i seaborn
  • ✅ Organizacja projektu: kod został podzielony na logiczne moduły, co ułatwia rozwój, testowanie i utrzymanie.
