Sklep Internetowy

System Zarządzania Zamówieniami

Opis Projektu

Projekt przedstawia kompletny system bazy danych dla sklepu internetowego, stworzony z myślą o praktycznym zastosowaniu wiedzy z zakresu SQL, T-SQL, optymalizacji oraz bezpieczeństwa danych. Zawiera pełen cykl projektowy: od modelu relacyjnego, przez implementację struktur i procedur, aż po zabezpieczenia i kopie zapasowe.

Użyte Technologie

  • SQL
  • T-SQL
  • Git

🧱 Struktura bazy danych

  • ✅ Tabele: Klienci, Produkty, Zamówienia, SzczegółyZamówienia, Płatności, Dostawy, Kategorie, HistoriaProduktów.
  • ✅ Typy danych dobrane zgodnie z przeznaczeniem.
  • ✅ Domyślne wartości i ograniczenia (np. CHECK, DEFAULT, UNIQUE).

⚙️ Procedury składowane

  • ✅ sp_DodajZamowienie – tworzenie zamówienia na podstawie danych w formacie JSON, z automatycznym obliczaniem wartości oraz aktualizacją stanu magazynowego.
  • ✅ sp_ZmienStatusZamowienia – aktualizacja statusu zamówienia z walidacją.
  • ✅ sp_RaportSprzedazy – generowanie raportu miesięcznej sprzedaży.
  • ✅ sp_ObslugaDostawy – automatyczne tworzenie rekordu dostawy dla zamówienia.
  • ✅ sp_TopKlienci – zestawienie klientów z największymi wydatkami.

🧠 Trigger

  • ✅ trg_LogowanieZmianProduktow – automatyczne rejestrowanie zmian cen produktów do tabeli historii (HistoriaProduktow), z datą i użytkownikiem, który zmiany dokonał.

⚡ Optymalizacja

  • ✅ Indeksy na często filtrowanych kolumnach (np. idKlienta, idZamowienia, idProduktu).
  • ✅ CHECK i DEFAULT zabezpieczające dane.

🔐 Bezpieczeństwo danych

  • ✅ Utworzenie kont SQL dedykowanych dla różnych typów użytkowników
  • ✅ Przypisanie im tylko tych uprawnień, które są niezbędne
  • ✅ Projekt wykorzystuje Dynamic Data Masking, aby zapobiec wyciekom danych osobowych.

💾 Backup i restore

  • ✅ Podczas tworzenia projektu wielkrotnie tworzono kopie zapasowe.
