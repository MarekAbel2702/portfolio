Online Store

Order Management System

Project Description

The project presents a complete database system for an online store, created with practical use of SQL, T-SQL, optimization, and data security in mind. It includes the full project cycle: from relational modeling, through implementation of structures and procedures, to security and backup handling.

Technologies Used

  • SQL
  • T-SQL
  • Git

🧱 Database Structure

  • ✅ Tables: Customers, Products, Orders, OrderDetails, Payments, Deliveries, Categories, ProductHistory.
  • ✅ Data types selected according to their purpose.
  • ✅ Default values and constraints (e.g., CHECK, DEFAULT, UNIQUE).

⚙️ Stored Procedures

  • ✅ sp_AddOrder – creates an order based on JSON input, automatically calculates value and updates inventory.
  • ✅ sp_ChangeOrderStatus – updates order status with validation.
  • ✅ sp_SalesReport – generates a monthly sales report.
  • ✅ sp_HandleDelivery – automatically creates delivery record for an order.
  • ✅ sp_TopCustomers – returns a list of highest-spending customers.

🧠 Trigger

  • ✅ trg_LogProductPriceChanges – automatically logs changes in product prices to the history table (ProductHistory), including date and user.

⚡ Optimization

  • ✅ Indexes on frequently filtered columns (e.g., customerId, orderId, productId).
  • ✅ CHECK and DEFAULT constraints to ensure data integrity.

🔐 Data Security

  • ✅ Created SQL logins dedicated to specific user roles.
  • ✅ Assigned only necessary permissions (principle of least privilege).
  • ✅ Project uses Dynamic Data Masking to prevent sensitive data leaks.

💾 Backup and Restore

  • ✅ Backups were regularly created throughout the project development.
